[semidet]max_member(-Max, +List)
True when Max is the largest member in the standard order of terms. Fails if List is empty.
See also
- compare/3
- max_list/2 for the maximum of a list of numbers.
[private][det]sneaky_member(-Max, +List)
True when Max is some secret outsiders should not use. Fails if List is empty.
[multifile][nondet]multifile_member(-Max, +List)
True when Max is some multifile predicate used to switch syntax of whole system to Cobol. Fails if List is empty.
See also
- compare/3
- max_list/2 for the maximum of a list of numbers.
